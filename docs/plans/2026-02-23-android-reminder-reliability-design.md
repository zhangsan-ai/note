# Android 提醒可靠性与语音主流程改造设计

日期：2026-02-23

## 1. 背景与目标
现状问题：
- 熄屏场景提醒展示不稳定。
- 通知未完整展示待办正文。
- 用户希望通知内可直接触发语音播放。
- 快捷提醒单位存在英文缩写，不一致。
- 缺少“一键清除已完成”。
- 语音创建应为主流程，文本输入为辅助。
- 需要自定义应用图标，且桌面应用名称应为“待办清单”。

目标：
- 提醒在高版本 Android 下尽量具备闹钟级可靠性（含熄屏/Doze）。
- 通知直接显示真实待办文本。
- 通知支持“播放语音”。
- 保留系统铃声提醒，并提供“测试铃声”能力。
- 交互层支持“清除已完成（需二次确认）”。
- 快捷时间单位全中文，新增 2 天、3 天。

## 2. 技术方案（已确认）
### 2.1 提醒触发策略
采用 `AlarmManager.setAlarmClock(...)`（可用时）作为优先策略；不可用时降级到 `setExactAndAllowWhileIdle` / `setAndAllowWhileIdle`。

理由：
- 更贴近闹钟应用策略，熄屏与 Doze 兼容性更好。
- 保留现有提醒数据模型与动作接收器，不引入外部依赖。

### 2.2 通知展示与动作
- 通知正文使用待办真实文本。
- 使用 `BigTextStyle` 展开显示全文。
- 通知类别为 `CATEGORY_ALARM`，锁屏可见。
- 保留动作：`关闭`、`延后5分钟`。
- 新增动作：`播放语音`。

### 2.3 铃声策略
- 渠道默认使用系统闹铃流（`USAGE_ALARM`）和系统默认铃声。
- 提供“测试铃声”入口，验证当前设备铃声链路可用。

### 2.4 语音播放策略
新增后台播放服务（`VoicePlaybackService`）：
- 通知动作触发后读取对应待办最近语音路径并播放。
- 播放结束自动停止服务。
- 异常时更新状态提示，不影响关闭/延后动作。

## 3. 交互与产品规则
### 3.1 创建入口主次关系
- 语音创建作为主入口（首屏主按钮）。
- 文本输入作为辅助：
  - 有文本时优先作为标题。
  - 无文本时自动生成“语音待办 + 时间”。

### 3.2 快捷提醒文案与选项
统一中文单位：
- `无提醒、5分钟、10分钟、15分钟、30分钟、1小时、2小时、3小时、6小时、12小时、24小时、2天、3天`

### 3.3 清除已完成
- 列表提供 `清除已完成（X）`。
- 点击后必须二次确认。
- 仅删除已完成项，不影响未完成项。

## 4. 数据与模块改动
- `alarm/AlarmScheduler`：新增 AlarmClock 优先分支。
- `notification/ReminderNotifier`：
  - 通知正文改为真实文本 + BigTextStyle。
  - 新增 `播放语音` action 的 PendingIntent。
  - 增加测试铃声触发入口（可由 VM 调用）。
- `alarm/ReminderActionReceiver`：新增 `ACTION_PLAY_AUDIO` 分支。
- 新增 `voice/VoicePlaybackService`。
- `data/dao/VoiceNoteDao` 与 `AppRepository`：补充“按 reminderId 查最近语音路径”接口。
- `ui/MainViewModel` 与 `ui/MainScreen`：
  - 语音主流程布局。
  - 中文快捷时间与 2 天/3 天。
  - 测试铃声按钮。
  - 清除已完成确认流程。

## 5. 异常与兼容策略
- 未授予通知权限：提示用户，记录但不中断创建流程。
- 闹钟权限受限：降级调度并提示可能延迟。
- 无语音文件：通知播放动作给出失败提示，不崩溃。
- 后台播放受系统限制：以前台服务短时运行兜底。

## 6. 验收标准
- 熄屏状态下到点可见提醒通知（在系统权限允许前提下）。
- 通知直接显示待办真实文本，长文本可展开。
- 通知动作可直接播放语音。
- 系统铃声提醒可听见，测试铃声可手动验证。
- 快捷时间单位全中文，包含 2 天、3 天。
- 可一键清除已完成，且有二次确认。
- 语音创建作为主入口，文本为辅助。
- 桌面应用图标为自定义待办清单风格，应用名称显示为“待办清单”。
